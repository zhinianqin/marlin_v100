# SM70 dense uint8b128 支持记录

## 背景

本轮为 dense SM70 CUTLASS 路径新增 `vllm::kU8B128` / `uint8b128` 支持。它和 `uint4b8` 一样没有 zero-point metadata，运行时数学是：

```text
dequant(q) = q - 128
weight = dequant(q) * scale
```

因此 `uint8b128` 只需要 `b_q_weight` 和 `b_scales`，不接受 `b_zp_bias`，也不允许 `use_zp_bias=true`。MoE 的 `uint8b128` 本轮不重新接入，support matrix 仍保持 `moe_supported=False`。

## 实现

- 新增 `csrc/quantization/marlin/sm70_marlin_u8b128_gemm.cu`，结构对齐 `sm70_marlin_u4b8_gemm.cu`。
- 几何固定为 `128x256x32 / 8 warp`，即 `kCtaM=128`、`kCtaN=256`、`kCtaK=32`、`kWarps=8`。
- `GroupSize=-1` 在 IteratorB constructor 中内联缓存 scale；`GroupSize=32/64/128` 在 `load()` 内、`load_full_tile()` / `load_residue_tile()` 外刷新当前 group scale cache。
- full tile 路径每个 B fragment access 读取两个 `uint4` 向量：
  - `qwords0` 覆盖 4 个 N64 subtile 的低 4 个 uint8。
  - `qwords1` 覆盖 4 个 N64 subtile 的高 4 个 uint8。
- hot path 使用 `dequant<half2, vllm::kU8B128.id(), false>` 和 `__hmul2`，不持有 `zp_bias_`，不生成 `cached_bias_`，不使用 `__hfma2`。
- residue 路径保留显式 invalid fragment 写 0，并加注释记录该路径接近寄存器上限，full tile 是主要热路径。

## qweight layout

`uint8b128` 复用当前 8-bit macro-N layout，与 dense `uint8` 完全一致：

```text
[k_tile][n256_macro][local_word][n64_subtile]
```

地址公式：

```text
k_tile = logical_k / 16
local_k = logical_k % 16
n_tile = logical_n / 64
macro_n_tile = n_tile / 4
subtile = n_tile % 4
subtile_count = min(4, size_n / 64 - macro_n_tile * 4)
local_n_word = (logical_n % 64) / 4
local_word = local_k * 16 + local_n_word

offset =
  k_tile * (size_n * 4) +
  macro_n_tile * 4 * 256 +
  local_word * subtile_count +
  subtile
```

其中 `256 = 16 * 64 / 4` 是单个 `16x64` int8 tile 的 `uint32` word 数量。full tile 下同一个 `local_word` 的 4 个 N64 subtile 连续，因此可以用 128-bit load 合并读取。

## 接口与支持矩阵

- `marlin.cu` 现在允许 dense `uint4 / uint4b8 / uint8 / uint8b128`。
- `uint8b128` 分支要求：
  - `b_zp_bias` 为空。
  - `use_zp_bias=false`。
  - `size_k % 32 == 0`。
  - `size_n % 64 == 0`。
  - `act_order` 不支持。
- `python/marlin_v100/calibration.py` 中 `uint8b128` 改为 `dense_supported=True, moe_supported=False`。
- dense benchmark 默认候选加入 `uint8b128`。

## 对齐结论

`diff -u sm70_marlin_u4b8_gemm.cu sm70_marlin_u8b128_gemm.cu` 的剩余差异属于必要功能差异：

- `u8b128` 类型/命名。
- 8-bit pack factor：`kU8ValuesPerWord=4`，`kU8ValuesPerAccess=8`。
- 每个 access 需要两个 qword，因此 full tile 使用两个 `uint4` load，residue 使用两个 scalar qword load。
- 8-bit macro-N 下第二个 qword 需要 `qweight_word_stride()`。
- dequant 使用 `vllm::kU8B128`，但后续仍是 `__hmul2(deq, scale)`，与 `u4b8` 保持无 zero metadata 语义。
