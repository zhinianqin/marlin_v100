# SM70 NVFP4 dense 预转换 scale 路径

## 背景

本轮新增 SM70 dense-only NVFP4 weight path：

- weight type: `vllm::kFE2M1f` / `float4_e2m1f`
- activation/output: `float16`
- group size: `16`
- scale input to SM70 direct kernel: preconverted `float16`
- global scale: single-element `float32`

这条路径不复制上游 generic Marlin NVFP4 的 hot-path FP8 scale decode。上游 generic path 使用 FP8 block scales 和 `global_scale`，并在模板主循环中调用 `dequant_fp8_scales(...)`。本地 SM70 direct path 参考 `sm70_marlin_u4_gemm.cu` 的 `b_zp_bias` 方式，把 scale 元数据提前处理成 kernel 可直接消费的 FP16。

## 数据流

Python/helper 侧生成：

- packed FP4 qweight: SM70 native int32 Marlin layout
- FP16 block scales: `fp8_scale.to(float16) * 128`
- FP32 global scale: `global_scale * 128`
- reference weight: `fp4_value * fp8_scale * global_scale`

kernel 侧只做：

- load packed FP4 qword
- `marlin::dequant<half2, vllm::kFE2M1f.id(), true>(...)`
- multiply preconverted FP16 scale
- Volta FP16 Tensor Core MMA
- epilogue alpha multiply by FP32 global scale

因此总补偿仍等价于 FP4-to-FP16 exponent bias 的 `2^14`，但被拆成 scale 侧 `2^7` 和 global scale 侧 `2^7`，避免把完整补偿塞进 FP16 scale。

## 支持边界

当前只支持 dense weight-only：

- `group_size == 16`
- `size_k % 32 == 0`
- `size_n % 64 == 0`
- no BF16
- no FP4/FP8 activation
- no MXFP4
- no MoE
- no act_order
- no bias
- no zero-point metadata
- no atomic add
- `is_k_full=True`

raw FP8 scales 直接传入 SM70 direct `marlin_gemm` 会报错；调用方必须走本地 helper，把 NVFP4 scales 预转换为 FP16，并同时传入 FP32 `global_scale`。
